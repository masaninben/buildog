# Buildog — CLAUDE.md

## プロダクト概要

**Buildog**（ビルドッグ）= Build + Log。
リフォーム・建築工事の施工写真を案件ごとに整理し、施主に分かりやすく共有するための軽量PWA。

**本質：施工管理ツールではなく「現場の安心を施主に見せるための共有ツール」**

- 写真の順番と構成を整えることが目的
- 施主はログイン不要・URLを開くだけ
- 「ちゃんとやってます」を写真で伝えるサービス

### Penstokとの関係

BuildogはPenstokとは**別プロダクト・別Firebaseプロジェクト**。
カード一覧・詳細・写真という構造は似ているが、概念・目的・ユーザー・データは別物。

## 技術スタック

- **フロントエンド**: Vue 3 + TypeScript 5.4 + Vite 5.3
- **バックエンド**: Firebase（Hosting / Firestore / Auth / Storage）
- **PDF生成**: jsPDF
- **QRコード**: qrcode
- **ホスティング**: Firebase Hosting（https://buildog-jp.web.app/）
- **リポジトリ**: https://github.com/masaninben/buildog
- **外部依存**: Firebase のみ（他サービス追加は原則禁止）

## ターゲットユーザー

### 業者側（課金者・ログインあり）
- リフォーム業者・工務店・外壁塗装業者・設備工事業者
- 営業担当・事務担当・現場担当

### 施主側（ログインなし・閲覧のみ）
- 工事を依頼した施主・家族・管理者
- 30〜70代、デジタル弱者（高齢者）含む

---

## 設計思想

1. **現場・営業・事務が迷わないこと** — 軽量で直感的、多機能施工管理ツールではない
2. **写真はデフォルト非公開** — アップロード後まず内部管理、公開写真だけ施主ページに表示
3. **写真の並び順が重要** — ビフォー→施工中→材料→アフター の流れが施主の理解を作る
4. **一覧性を重視** — カードは小さめ・多く見えるように
5. **スマホ優先** — 現場はスマホ撮影、事務はPCで一括整理

- ビフォー → 施工中 → 材料 → アフター の順に表示

## タグ体系

| タグ | 用途 |
|---|---|
| ビフォー | 工事前の状態 |
| 施工中 | 工事中（「工程」表記は使わない） |
| 材料 | 使用塗料・設備・資材 |
| アフター | 工事後の仕上がり |
| タグ未設定 | デフォルト（未分類） |

---

## データ構造

```
Firestore:
projects/{projectId}
  - name, clientName, siteAddress
  - ownerId, createdAt, updatedAt
  - isPublic, publicSlug
  - coverPhotoUrl, coverPhotoId

projects/{projectId}/photos/{photoId}
  - url, storagePath, createdAt, uploadedBy
  - isPublic, tag, memo, sortOrder

Storage:
  projects/{projectId}/{photoId}.jpg
```

## UI方針

- ダークテーマ（ネイビー / ブラック / ブルー / オレンジ）
- 角丸カード・写真主役
- 余白は取りすぎず一覧性重視
- 切り替えはトグルスイッチ
- タグはチップUI
- ボタンは最小限

### ブルドッグキャラクター
- 「工事の証拠を守る番犬」
- UI上では主役でなく案内役
- 枠付きアイコンではなく輪郭で切り抜いた画像として使う
- `/src/assets/characters/` に背景透過PNG・SVG両方を用意（SVG優先）

## 開発方針

- 既存機能を壊さない・大規模リファクタしない
- 実装済みならスキップ
- 機能を増やすより「見せ方の質」を上げることを優先
- コンポーネントは小さく保ち、Composables で状態管理
- Firebase 以外の外部依存は原則追加しない
