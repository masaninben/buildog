# 画面仕様

## ルーティング一覧

```
/                   ホーム（タイムライン）     認証必須
/equipment          設備一覧                  認証必須
/equipment/new      設備登録                  認証必須
/equipment/:id      設備詳細                  認証必須
/trouble            困り事相談TOP             認証必須
/trouble/category   カテゴリ選択              認証必須
/trouble/form       困り事入力                認証必須
/trouble/result     おすすめ業者表示          認証必須
/profile            家の基本情報              認証必須
/settings           設定                      認証必須
/login              ログイン                  未認証のみ
```

---

## 共通レイアウト

### AppHeader
- 左：ハンバーガーメニュー（≡）
- 中央：画面タイトル または「Buildog Karte」ロゴ
- 右：フクロウアイコン（装飾、タップ不可）

### AppSidebar（左寄せ、オーバーレイ）
- オーバーレイ背景タップで閉じる
- メニュー項目：
  - ホーム
  - 設備一覧
  - 書類管理（Phase2以降）
  - 家の基本情報
  - 設定
  - ログアウト

### BottomNav（5タブ固定）
```
🏠 ホーム  /  📋 タイムライン  /  🔧 設備  /  ❓ 困り事  /  ⚙️ 設定
```
現在の画面に対応するタブをアクティブ表示。

### FloatingButton（右下固定）
- ＋ボタン
- ホーム・タイムラインでのみ表示
- タップでイベント追加モーダル（写真を追加 / メモを書く / 工事を記録）

## ホーム画面（タイムライン）`/`

### 概要
家の歴史を時系列で表示する。新しいイベントが上、古いイベントが下。
新築・購入時が最下部の起点になる。

### タイムライン表示仕様

```
2026年（令和8年）
┃
●  2月12日
   太陽光発電設置                          [詳細 >]
┃
2025年（令和7年）
┃
●  6月11日
   外壁塗装                                [詳細 >]
┃
1990年（平成2年）
┃
●  3月22日
   入居開始                                [詳細 >]
┗━（起点）
```

- 年が変わるタイミングに年表示（西暦 + 和暦）を挿入
- 左端に縦線＋丸のタイムライン軸
- 詳細ボタンは小さく右端に配置
- イベントタイプ別にアイコンを左の丸に表示

### イベントタイプアイコン
```
move_in        🏠
construction   🔨
repair         🔧
inspection     🔍
replacement    🔄
purchase       🛒
memo           📝
photo          📷
other          📌
```

### 空状態
設備・イベントがない場合はフクロウ表示。
「家の情報を登録してみましょう」→ 家の基本情報登録へ誘導。

## 設備一覧 `/equipment`

### 概要
登録済み設備をカードグリッドで表示。カテゴリ別にグループ化。

### カード表示項目
- カテゴリアイコン（大）
- 設備名
- 状態ラベル（色付きバッジ）
- 設置年（あれば）
- 保証期限（90日以内なら赤表示）

### グリッドレイアウト
- スマホ：2列
- タブレット以上：3〜4列

### 状態ラベル色
```
good          緑  #2D6A4F
check_needed  橙  #E07B39
warranty_soon 黄  #D4A017
replace_soon  赤  #C0392B
unknown       灰  #999999
```

### 空状態
フクロウ表示。「設備を登録してみましょう」→ 設備登録へ誘導。

---

## 設備登録 `/equipment/new`

### ステップUI（5ステップ）

**Step 1：カテゴリ選択**
- グリッド選択UI（アイコン＋名前）
- 17カテゴリ全表示

**Step 2：写真追加**
- 任意
- 「あとで追加できます」を明示
- カメラ撮影 / ライブラリから選択

**Step 3：基本情報**
- 設備名（任意、カテゴリから自動入力可）
- メーカー（任意）
- 型番（任意）

**Step 4：日付情報**
- 設置日（任意）
- 保証期限（任意）
- メモ（任意）

**Step 5：確認・保存**

### UX原則
- 全項目任意
- 「あとで入力」を許可
- 写真だけでもOK
- ステップを前後に移動可能

---

## 設備詳細 `/equipment/:id`

### セクション構成（上から順）

1. **メインカード**
   - 設備写真（スライド表示）
   - 設備名
   - カテゴリ
   - 状態ラベル
   - メーカー / 型番
   - 設置日 / 保証期限

2. **アクションボタン行**
   - 記録を追加
   - 修理履歴を追加
   - 保証書を追加
   - 困り事を相談

3. **コンテキスト提案**（条件付き表示）
   - 表示条件：installedDate あり かつ 経過年数 >= 寿命 × 0.8
   - 表示内容：「設置からN年が経ちました。点検の相談ができます。」
   - ボタン：「近くの業者を見る」→ /trouble/form へ（カテゴリ自動設定）
   - **設備詳細画面のみで表示。他画面には出さない。**

4. **履歴リスト**
   - 点検・修理・交換・写真・メモを時系列表示

5. **関連書類**
   - 保証書 / 取扱説明書 / 見積書 / 契約書

## 困り事相談 `/trouble`

### フロー

```
/trouble （TOP）
  ↓
/trouble/category （カテゴリ選択）
  ↓
/trouble/form （状況入力）
  ↓
/trouble/result （おすすめ業者表示）
```

### カテゴリ選択
設備カテゴリコードと同一のグリッドUI。

### 状況入力フォーム
- 状況説明（テキスト）
- 写真（任意）
- 緊急度（urgent / soon / research）
- 希望エリア（areaLabel形式、デフォルト：家の登録エリア）

### 業者表示（MVP）
- MVPではモックデータ or Firestoreに手動登録した業者データを表示
- 実装後に課金連携

## 家の基本情報 `/profile`

### 入力項目
- 住宅名（例: ○○邸）必須
- 所在エリア（例: 福岡県福岡市）必須
  - 詳細住所は不要、市区町村レベルでOK
- 入居日（任意）
- 建築年（任意）
- 建物種別（任意）：戸建 / マンション / etc
- 構造（任意）：木造 / RC / 鉄骨 / etc
- 延床面積（任意）

### 初回登録
ログイン後にhomesコレクションが空の場合、プロフィール登録を促す。

## 設定 `/settings`

### 項目
- アカウント情報（表示名・メール）
- 家情報の編集 → /profile へ
- 通知設定（将来）
- データ連携（Buildog連携状態表示）
- ログアウト
- 退会

---

## ログイン `/login`

### 内容
- Googleでログインボタン（一択）
- アプリの説明（1〜2行）
- フクロウイラスト

### 認証後
- homesコレクションにデータあり → / へリダイレクト
- homesコレクションが空 → /profile（初回登録）へリダイレクト
